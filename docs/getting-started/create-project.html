Create a Kitura project

There's several ways to create a Kitura project:

  • SPM: The Swift Package Manager is the standard way of creating Swift on the server projects.
  • Kitura CLI: A command line interface to make creating a Kitura project easier and faster.
  • Kitura macOS App: A UI based version of the Kitura CLI. Ideal for those wishing to avoid the terminal.

Create a project with SPM

Step 1: Create project

Create a directory for your project:

mkdir HelloWorld && cd HelloWorld

Inialize the directory as a Swift project:

swift package init --type executable

Step 2: Add Kitura to your dependencies

To add Kitura to your dependencies you need to edit the Package.swift

Open Package.swift in your default text editor:

open Package.swift

Add Kitura to your dependencies so your Package.swift looks like this:

// swift-tools-version:4.2
// The swift-tools-version declares the minimum version of Swift required to build this package.

import PackageDescription

let package = Package(
    name: "spm-example",
    dependencies: [
        // Dependencies declare other packages that this package depends on.
        // .package(url: /* package url */, from: "1.0.0"),
        .package(url: "https://github.com/IBM-Swift/Kitura", from: "2.5.0")
    ],
    targets: [
        // Targets are the basic building blocks of a package. A target can define a module or a test suite.
        // Targets can depend on other targets in this package, and on products in packages which this package depends on.
        .target(
            name: "spm-example",
            dependencies: ["Kitura"]),
        .testTarget(
            name: "spm-exampleTests",
            dependencies: ["spm-example"]),
    ]
)

Now build your project to pull down your dependencies:

swift build

Create a project with Kitura CLI

Step 1: Create a directory

Kitura CLI needs to be run in an empty directory so you will need to create a new directory for your project:

mkdir KituraExample && cd KituraExample

Step 2: Create a Kitura project

Create a Kitura application using the Kitura CLI:

kitura init

`kitura init` will create a cloud ready Kitura application in your current directory and also builds your application.

The name of your project will be the same as the directory name.
