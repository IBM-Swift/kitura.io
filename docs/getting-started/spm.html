Swift Package Manager

The Swift Package Manager, or SPM, provides a way to create Swift libraries and Swift apps on macOS and Linux.

SPM also provides a way for you to manage your dependencies and makes sharing Swift code easier.

Kitura is a Swift code base which is made usuable by SPM, that is you will use SPM to add Kitura to a Swift project.

Creating a Kitura project

Before continuing with these steps ensure you have setup your workspace.

Step 1: Initialize project

swift package init --type executable

Step 2: Add Kitura to project

To add a new dependency to your project you need to modify the Package.swift file.

Firstly in a terminal open the Package.swift with your default text editer

open Package.swift

Next add the following line to the dependencies section:

.package(url: "https://github.com/IBM-Swift/Kitura", from: "x.x.x")

Where 'x.x.x' is your chosen version. You can find a list of the releases on Github.

Finally you need to add 'Kitura' to your target's dependencies.

Your Package.swift should look something like this:

// swift-tools-version:4.2
// The swift-tools-version declares the minimum version of Swift required to build this package.

import PackageDescription

let package = Package(
    name: "spm-example",
    dependencies: [
        // Dependencies declare other packages that this package depends on.
        // .package(url: /* package url */, from: "1.0.0"),
        .package(url: "https://github.com/IBM-Swift/Kitura", from: "2.5.0")
    ],
    targets: [
        // Targets are the basic building blocks of a package. A target can define a module or a test suite.
        // Targets can depend on other targets in this package, and on products in packages which this package depends on.
        .target(
            name: "spm-example",
            dependencies: ["Kitura"]),
        .testTarget(
            name: "spm-exampleTests",
            dependencies: ["spm-example"]),
    ]
)

Step 3: Build the project

In a terminal run:

swift build

Once this has you can create an executable to run:

swift run

This will output an executable for you to run. Copy and paste the executable to your terminal to start your server.

Step 4: Run in Xcode

If you're using Xcode you can also generate an Xcode project:

swift package generate xcode-proj

This will generate an Xcode project for you allowing you to develop your server within Xcode.
