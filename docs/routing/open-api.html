Kitura OpenAPI

Prerequisites

  • Kitura Server: Learn how to create one in our Getting Started guide
  • Codable Routes: Learn how to add Codable routes in our Codable Routing guide

Step 1: Add Kitura OpenAPI to your project

First we need to import KituraOpenAPI into our project:

import KituraOpenAPI

Next we need to add the KituraOpenAPI end points to our router.

Just above the `Kitura.addHTTPServer()` line, add:

KituraOpenAPI.addEndpoints(to: router)

That's all we need to do to enable KituraOpenAPI on our server.

Step 2: View the OpenAPI UI

We need to start our Kitura server first.

Once the server has started, navigate to:

localhost:8080/openapi/ui

You should see your Codable routes listed:

We can then click on the list routes to find out more information about each individual route:

From here we can see the response code from the server which is 200 (Success) in our case

We can also see an example value in JSON for our User.

Finally we can actually test our route from this UI by hitting the `Try it out` button:

As this is a get route we don't need to provide any data so just hit execute:
